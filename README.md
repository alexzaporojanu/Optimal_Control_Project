# Optimal Control of an Acrobot

Trajectory optimization and tracking control for an underactuated two-link planar robot (Acrobot) using Newton's method, TV-LQR, and Model Predictive Control.

## System Description

The **Acrobot** is a two-link pendulum actuated only at the elbow joint (joint 2). The shoulder joint (joint 1) is unactuated, making the system **underactuated**. The state vector is:

```
x = [θ₁, θ₂, dθ₁, dθ₂]ᵀ ∈ ℝ⁴
```

where θ₁ is the shoulder angle (measured from the downward vertical) and θ₂ is the relative elbow angle. The single control input is the elbow torque τ ∈ ℝ.

### Equations of Motion

The dynamics follow the standard Euler-Lagrange formulation:

```
M(q)q̈ + C(q, q̇)q̇ + Fq̇ + G(q) = [0; τ]
```

where:
- **M(q)** — symmetric positive-definite mass/inertia matrix (2×2)
- **C(q, q̇)** — Coriolis and centrifugal matrix (2×2)
- **F** — viscous friction matrix (diagonal)
- **G(q)** — gravitational torque vector

The continuous dynamics are discretized via a **4th-order Runge-Kutta (RK4)** integrator with time step `dt = 0.01s`. Jacobians of the discrete map are computed via **CasADi algorithmic differentiation** (exact, not finite differences).

### Physical Parameters (Set 3)

| Parameter | Value | Unit |
|-----------|-------|------|
| Link masses (m₁, m₂) | 1.5, 1.5 | kg |
| Link lengths (l₁, l₂) | 2.0, 2.0 | m |
| CoM distances (lc₁, lc₂) | 1.0, 1.0 | m |
| Inertias (I₁, I₂) | 2.0, 2.0 | kg·m² |
| Viscous friction (f₁, f₂) | 1.0, 1.0 | N·m·s |
| Gravity (g) | 9.81 | m/s² |

## Tasks

### Task 0 — Passive Dynamics Validation

Simulates the Acrobot under zero torque from a horizontal initial condition. Validates that the Euler-Lagrange equations and RK4 integration produce physically plausible behavior (energy dissipation via friction, pendulum-like oscillations).

**Run:** `python3 task0_main.py`

### Task 1 — Trajectory Optimization (Step Reference)

Computes an optimal swing-up trajectory from the downward equilibrium ($\theta_1 = 0$, $\theta_2 = -\pi$) to the upward equilibrium ($\theta_1 = 0$, $\theta_2 = \pi$) using a **regularized Newton's method** (Sequential Linear Quadratic / iLQR). The reference trajectory is a discontinuous step that switches at $T/2 = 5.0\text{ s}$.

**Algorithm:**
1. Linearize the nonlinear dynamics along the current trajectory
2. Compute the search direction via the **Affine LTV-LQR** backward Riccati recursion using the Traditional Riccati Formulation.
3. Determine the step size via **Armijo backtracking** line search
4. Update the trajectory using the closed-loop Newton rule:
   ```
   u_t^{k+1} = u_t^k + K_t * (x̂_t - x_t^k) + γ * σ_t
   ```
5. Propagate states through the nonlinear dynamics

**Run:** `python3 task1_main.py`

### Task 2 — Trajectory Optimization (Smooth Reference)

Same optimization framework as Task 1, but with a **C² smooth quintic Hermite** reference trajectory:

```
s(τ) = 10τ³ - 15τ⁴ + 6τ⁵,   τ ∈ [0, 1]
```

The trajectory is structured in three phases:
1. **Pre-wait** (5s) — hold at downward equilibrium
2. **Smooth move** (10s) — quintic transition to upward equilibrium
3. **Post-hold** (5s) — stabilize at upward equilibrium

The initial guess is generated by simulating the open-loop system dynamics starting from the initial downward equilibrium using the reference control input trajectory.

**Run:** `python3 task2_main.py`

### Task 3 — Trajectory Tracking via TV-LQR

Tracks the optimal trajectory from Task 2 using a **Time-Varying Linear Quadratic Regulator (TV-LQR)**. The system is linearized along the reference, and the backward Riccati equation (implemented in traditional form) yields time-varying feedback gains $K_t$.

The tracking law applies a deviation-feedback correction:
```
u_t = u_t* - K_t * (x_t - x_t*)
```

Tested initial state perturbations include:
1. **Shoulder joint error:** $\Delta x_0 = [-0.4, 0.0, 0.0, 0.0]^\top$ ($-22.92^\circ$)
2. **Elbow joint error:** $\Delta x_0 = [0.0, 0.3, 0.0, 0.0]^\top$ ($+17.19^\circ$)
3. **Shoulder velocity error:** $\Delta x_0 = [0.0, 0.0, 0.3, 0.0]^\top\text{ rad/s}$

**Run:** `python3 task3_main.py` (requires Task 2 output)

### Task 4 — Trajectory Tracking via MPC

Tracks the optimal trajectory using **Model Predictive Control (MPC)** with CasADi/IPOPT. At each time step, a finite-horizon LTV optimization is solved in deviation variables with:

- Linear dynamics constraints from the reference linearization
- Absolute torque bounds ($|u| \leq 30.0\text{ Nm}$)
- Riccati terminal cost from Task 3 for closed-loop stability

Tested initial state perturbations include:
1. **Shoulder joint error:** $\Delta x_0 = [-0.2, 0.0, 0.0, 0.0]^\top$ ($-11.46^\circ$)
2. **Elbow joint error:** $\Delta x_0 = [0.0, 0.3, 0.0, 0.0]^\top$ ($+17.19^\circ$)

**Run:** `python3 task4_main.py` (requires Task 2 and Task 3 output)

## Execution Order

The task dependencies are structured as follows:

- **Verification Scripts (Standalone):**
  - `task0_main.py` validates the passive Acrobot dynamics (free fall).
  - `equilibrium_finding.py` validates the SQP root-finding of the discrete-time RK4 equilibria.

- **Trajectory Generation (Step vs. Smooth Reference):**
  - `task1_main.py` generates the optimal trajectory for the discontinuous step reference (`data/optimal_trajectory_task1.npy`).
  - `task2_main.py` generates the optimal trajectory for the smooth quintic reference (`data/optimal_trajectory_task2.npy`).

- **Trajectory Tracking (requires Task 2):**
  - `task3_main.py` tracks the smooth trajectory via Time-Varying LQR, saving gains and Riccati matrices (`data/lqr_data_task3.npy`).
  - `task4_main.py` tracks the smooth trajectory via Model Predictive Control. Note that Task 4 requires the terminal cost matrices generated by Task 3 (`data/lqr_data_task3.npy`).

**Recommended Execution Flow:**

To run the full trajectory generation and tracking pipeline:

1. Generate the smooth reference trajectory:
   ```bash
   python3 task2_main.py
   ```
2. Run the tracking controller of your choice:
   * **Option A: Time-Varying LQR Tracking:**
     ```bash
     python3 task3_main.py
     ```
   * **Option B: Constrained LTV-MPC Tracking** (requires running Task 3 first to generate terminal costs):
     ```bash
     python3 task3_main.py
     python3 task4_main.py
     ```

For step reference trajectory generation:
```bash
python3 task1_main.py
```

## Solver Hyperparameters

Key tuning parameters are centralized in `data.py`:

| Parameter | Value | Description |
|-----------|-------|-------------|
| `dt` | 0.01 s | RK4 discretization step |
| `max_iters` | 100 | Maximum Newton iterations |
| `term_cond` | 1e-4 | Convergence tolerance on descent |
| `armijo_c` | 0.5 | Sufficient decrease parameter |
| `armijo_beta` | 0.7 | Step contraction factor |
| `armijo_maxiters` | 20 | Maximum backtracking iterations |

## Output

Each task generates:
- **Convergence plots** — cost and descent metric vs iteration
- **Trajectory plots** — optimal states and inputs vs reference
- **Intermediate trajectory evolution** — visualization of convergence across iterations
- **Animations** — 2D forward kinematics visualization of the Acrobot motion
- **`.npy` data files** — serialized trajectories and gains for downstream tasks
